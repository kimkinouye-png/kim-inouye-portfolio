# Figma Make Export — Capacity Planner Case Study

This folder contains everything you need to work on the Capacity Planner case study in Figma Make and bring changes back to Cursor.

---

## What's in this folder

| File | Purpose |
|------|---------|
| **00-START-HERE.md** | This file — overview and quick start |
| **FIGMA-MAKE-CONTEXT.md** | Copy and paste into Figma Make as context so it understands the design |
| **design-spec.json** | Structured design tokens (colors, typography, layout) for reference |
| **capacity-planner-standalone.html** | Self-contained HTML preview — open in a browser to see the design |
| **images/** | Screenshots used in the case study (scenarios-comparison.png, capacity-dashboard.png) |
| **HANDOFF-BACK-TO-CURSOR.md** | Instructions for transferring your Figma changes back to Cursor |

---

## Quick start

### 1. Provide Figma Make with context

- Open **FIGMA-MAKE-CONTEXT.md**
- Copy the full content (everything under "What you're looking at")
- Paste it into Figma Make when it asks for context or a design brief

### 2. Use the standalone HTML as reference

- Open **capacity-planner-standalone.html** in a browser (Chrome, Safari, etc.)
- Toggle dark mode with the "Dark | Light" button
- Use this as a live reference while designing in Figma
- Take screenshots from here if Figma Make needs image references

### 3. After editing in Figma Make

- Follow **HANDOFF-BACK-TO-CURSOR.md** to apply your changes back to the Cursor project
- The source files in Cursor are: `capacity-planner.html`, `styles.css`, `images/`

---

## Download this folder

To use this package elsewhere (e.g. on another machine or share with a designer):

1. Zip the `figma-make-export` folder
2. The standalone HTML will work as long as the `images` subfolder is included
3. Unzip and open `capacity-planner-standalone.html` in a browser to verify
